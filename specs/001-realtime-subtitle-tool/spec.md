# Feature Specification: Windows 实时字幕工具

**Feature Branch**: `001-realtime-subtitle-tool`

**Created**: 2026-06-30

**Status**: Draft

**Input**: User description: "基于《基于 Tauri v2 与阶跃星辰的实时字幕工具开发架构》文档，构建一款 Windows 桌面级实时语音字幕工具。仅支持 Windows 平台，捕获系统播放器输出的音频（不采集麦克风），通过端到端实时语音大模型进行流式识别，以悬浮透明窗口形式在屏幕上层渲染实时字幕，并将历史字幕与副语言元数据持久化到本地数据库，供后续检索与分析。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 观看视频时获得实时字幕覆盖 (Priority: P1)

用户在 Windows 上使用任意播放器（浏览器视频、本地播放器、在线会议软件、播客应用等）播放音频/视频时，启动本工具后，屏幕上方会出现一个无边框、半透明的悬浮字幕条，随播放内容实时滚动显示识别出的文字。字幕窗口始终保持在最上层，不遮挡也不干扰用户对底层播放器的鼠标/键盘操作。当一句话讲完（出现自然停顿）后，该句被定格归档为最终字幕，新的草稿句继续在下方滚动出现。

**Why this priority**: 这是工具的核心价值入口——让用户在任意播放器场景下「看得到正在说什么」。其余所有故事（历史检索、样式定制、监控）都建立在该能力之上。该故事独立可用即构成 MVP。

**Independent Test**: 在 Windows 上启动工具 + 任意播放器播放一段带人声的音视频，观察悬浮窗是否在 2 秒内开始滚动出现文字，句尾停顿后是否归档为最终句，鼠标点击是否能穿透字幕窗到达下方播放器。

**Acceptance Scenarios**:

1. **Given** 工具已启动且播放器正在播放中文/英文语音内容，**When** 用户开启字幕开关，**Then** 悬浮窗在 2 秒内开始显示动态变化的草稿字幕，并在句尾停顿后定格为最终字幕
2. **Given** 字幕窗已显示在屏幕上方，**When** 用户用鼠标点击字幕窗覆盖区域的播放器控件，**Then** 鼠标事件穿透字幕窗，播放器正常响应（暂停/拖动进度条等）
3. **Given** 用户切换到全屏播放或最大化其他应用窗口，**When** 字幕窗仍处于置顶状态，**Then** 字幕始终可见，不被其他窗口遮挡
4. **Given** 播放器播放一段静音或纯背景音乐的片段，**When** 模型侧 VAD 判定无人声，**Then** 字幕窗不产生伪字幕，保持上一句状态或空白
5. **Given** 播放器音频中包含 DRM 保护内容，**When** 系统无法捕获有效音频流，**Then** 工具以可见方式提示用户「该内容受保护，无法识别」，而非静默失败

---

### User Story 2 - 检索与回看历史字幕 (Priority: P2)

用户在一次观看/会议结束后，希望回看本次会话的全部字幕记录，或按关键词检索过往会话中提到过的某句话。工具将每一次「从开启到关闭」视为一个会话，用户可在历史面板中选择某次会话查看完整时间轴字幕，也可输入关键词跨所有会话检索匹配句子。每条记录除文字本身外，还附带当时模型识别到的副语言信息（如情绪、语速、笑声/叹息等）。

**Why this priority**: 字幕的即时价值在于「当下听到」，但持久价值在于「事后找到」。本故事把转瞬即逝的语音转化为可检索的结构化语料，是工具从「辅助观看」升级为「内容存档」的关键。依赖故事 1 已产生的稳态字幕数据。

**Independent Test**: 完成一次 30 秒以上的播放识别后关闭再重开工具，进入历史面板，确认本次会话被记录；输入其中一个关键词，确认能命中并高亮显示对应句子及其所属会话。

**Acceptance Scenarios**:

1. **Given** 用户已完成一次会话并关闭工具，**When** 用户重新打开工具进入历史面板，**Then** 列表中出现该会话及其起止时间、时长
2. **Given** 历史面板已打开，**When** 用户选择某次会话，**Then** 按时间顺序展示该会话全部最终字幕，每条附带起止时间戳与副语言标签（如有）
3. **Given** 用户在搜索框输入关键词，**When** 触发检索，**Then** 跨所有历史会话返回包含该关键词的句子列表，命中关键词高亮，并显示所属会话与时间
4. **Given** 历史数据量较大，**When** 用户滚动列表，**Then** 列表保持流畅，无明显卡顿
5. **Given** 用户希望清理数据，**When** 选择删除某次会话或全部历史，**Then** 对应记录从列表与底层存储中彻底移除

---

### User Story 3 - 调整字幕外观与位置 (Priority: P3)

用户希望根据个人偏好和当前屏幕环境调整字幕的字体、字号、文字颜色、底板透明度，以及把字幕窗拖到屏幕任意位置（如底部居中、顶部居中、侧边）。位置与样式在工具重启后保持记忆。

**Why this priority**: 体验性优化。不同播放器、不同分辨率、不同视力条件下用户对字幕呈现的偏好差异大，但默认配置已可使用，故列为 P3。

**Independent Test**: 修改字号与底板透明度，拖动字幕窗到屏幕底部，关闭工具再重启，确认样式与位置被保留。

**Acceptance Scenarios**:

1. **Given** 字幕窗已显示，**When** 用户拖动字幕窗，**Then** 窗口可被移动到屏幕任意位置且不脱离屏幕可见区域
2. **Given** 用户在设置面板调整字号/颜色/底板透明度，**When** 应用更改，**Then** 字幕窗实时反映新样式
3. **Given** 用户已调整样式与位置，**When** 关闭并重新启动工具，**Then** 上次的样式与位置被恢复
4. **Given** 用户希望恢复默认，**When** 点击「重置为默认」，**Then** 样式与位置回到出厂配置

---

### Edge Cases

- **默认输出设备变更**：用户在播放过程中插拔蓝牙耳机/USB 声卡，系统默认输出设备切换。工具应能感知并自动重新绑定到新的默认输出设备，字幕中断不超过 3 秒，无需手动重启。
- **播放器无音频输出**：用户开启了字幕但播放器处于暂停或静音状态。工具不应崩溃或刷出伪字幕，应保持静默等待。
- **网络中断**：识别依赖的云端模型连接断开。工具应自动尝试重连（递增间隔），断连期间本地继续捕获音频并在内存中短暂缓存，重连后补发；若长时间无法重连，应向用户可见地提示「连接异常」，而非假装在识别。
- **API Key 无效或未配置**：用户首次使用未配置密钥，或密钥过期。工具应在启动识别时给出明确提示并引导配置，不进行静默失败。
- **长时间连续运行**：用户长时间（数小时）保持识别开启。工具内存占用不应持续爬升，日志文件不应无限增长。
- **多播放器同时输出**：用户同时开启多个播放器。工具捕获的是系统混音后的输出，字幕会合并识别多路人声——这是预期行为，应在文档/提示中说明。
- **极短句与极长句**：模型侧 VAD 对极短语音（如「嗯」「啊」）可能不下发稳态结果；对极长无停顿语音可能延迟下发。字幕窗应保持稳定，不因结果缺失而抖动。
- **DRM 保护内容**：受保护音频流无法被回环捕获，工具应识别该情况并提示用户，不输出乱码字幕。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 在 Windows 10/11 上运行，捕获系统默认输出设备（扬声器/耳机）的播放器音频，不采集麦克风输入。
- **FR-002**: 系统 MUST 以悬浮、无边框、半透明、始终置顶的窗口形式在屏幕上层显示字幕。
- **FR-003**: 系统 MUST 让鼠标点击/滚动事件穿透字幕窗，使底层播放器可正常交互。
- **FR-004**: 系统 MUST 在字幕窗内区分展示「正在变化的草稿句」与「已定格的最终句」，最终句归档至历史区。
- **FR-005**: 系统 MUST 在播放器音频出现自然停顿时，将当前草稿句定格为最终句，并附起止时间戳。
- **FR-006**: 系统 MUST 在云端识别连接断开时自动重连，断连期间本地缓存音频，重连后补发，并向用户可见地提示连接状态。
- **FR-007**: 系统 MUST 将每次「从开启到关闭」识别为一个会话，会话标识唯一；最终句及其时间戳、副语言信息持久化到本地。
- **FR-008**: 系统 MUST 提供历史面板，按会话列出过往记录，支持选择会话查看完整时间轴字幕。
- **FR-009**: 系统 MUST 提供跨会话的关键词检索能力，命中结果高亮并显示所属会话与时间。
- **FR-010**: 系统 MUST 允许用户删除指定会话或全部历史。
- **FR-011**: 系统 MUST 允许用户配置字幕样式（字体、字号、文字颜色、底板透明度），配置实时生效。
- **FR-012**: 系统 MUST 允许用户拖动字幕窗到屏幕任意位置，位置与样式在重启后保留。
- **FR-013**: 系统 MUST 在用户未配置或密钥无效时，给出明确提示并引导配置，不进行静默失败。
- **FR-014**: 系统 MUST 监听系统默认输出设备变更，自动切换捕获设备，中断不超过 3 秒。
- **FR-015**: 系统 MUST 仅将音频数据发送至用户配置的语音识别服务，历史数据与日志仅存储在本地，不上传任何第三方。
- **FR-016**: 系统 MUST 在识别到 DRM 保护内容无法捕获时，向用户提示「该内容受保护，无法识别」。
- **FR-017**: 系统 MUST 提供启动/停止字幕的开关控制，停止时优雅释放音频与网络资源。
- **FR-018**: 系统 MUST 在长时间运行时启用日志轮转，避免日志文件无限增长挤占磁盘。

### Key Entities *(include if feature involves data)*

- **会话 (Session)**: 一次从开启字幕到关闭的连续识别周期。属性：唯一标识、起止时间、所属最终字幕集合。
- **字幕条 (Transcription)**: 一句由模型下发并定格的最终识别结果。属性：所属会话、文本内容、起始时间戳、结束时间戳、副语言元数据（情绪/语速/非语言事件等，可为空）。
- **字幕样式配置 (AppearanceConfig)**: 用户偏好。属性：字体、字号、文字颜色、底板透明度、窗口位置。与用户一一对应，可重置为默认。
- **连接状态 (ConnectionStatus)**: 当前与识别服务的连接健康度。属性：是否已连接、累计重试次数、最近错误信息。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户开启字幕后，从播放器发出人声到字幕窗出现对应草稿文字，端到端延迟（不含模型固有处理时间）≤ 2 秒。
- **SC-002**: 在连续 30 分钟的播放识别过程中，字幕窗无崩溃、无冻结，内存占用稳定（增长幅度 ≤ 初始占用的 30%）。
- **SC-003**: 任意播放器场景下，鼠标点击字幕窗覆盖区域，100% 的事件穿透到底层播放器，播放器响应符合直接点击预期。
- **SC-004**: 用户在历史面板中按关键词检索，95% 的查询在 1 秒内返回结果，结果准确率 100%（即包含关键词的句子全部命中）。
- **SC-005**: 网络瞬时中断恢复后，断连期间的音频识别结果不丢失，字幕在重连后 5 秒内补齐显示。
- **SC-006**: 首次使用用户无需查阅外部文档即可完成「安装 → 配置密钥 → 开启字幕 → 看到识别结果」主流程，耗时 ≤ 5 分钟。
- **SC-007**: 工具在后台连续运行 8 小时，日志文件总占用磁盘 ≤ 50 MB。
- **SC-008**: 系统默认输出设备切换后，字幕中断恢复时间 ≤ 3 秒，且无需用户手动重启工具。

## Assumptions

- 用户运行环境为 Windows 10 或 Windows 11（x64），不提供 macOS / Linux 构建。
- 用户具备有效的阶跃星辰 StepAudio 实时识别服务 API Key，并通过应用内配置或环境变量提供。
- 音频源限定为系统播放器输出（系统混音后的回环音频），不采集麦克风；多播放器同时输出会被合并识别，此为预期行为。
- 用户已对「屏幕上方出现置顶悬浮窗」「事件穿透」等交互形态有基础接受度（参考 Wispr Flow 类产品）。
- 识别精度与延迟主要取决于云端模型，本工具负责把音频 faithfully 送达并渲染返回结果，不对模型自身准确率负责。
- 历史数据全部存储在用户本地，不提供云端同步与跨设备共享（v1 范围外）。
- 字幕样式配置面向单用户单设备，不涉及多账户体系。
- 受 DRM 保护的音频内容无法被回环捕获，这是平台限制而非工具缺陷，需在用户文档中明示。
