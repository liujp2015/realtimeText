# Quickstart 验证指南：Windows 实时字幕工具

**Date**: 2026-06-30
**Last Updated**: 2026-06-30（适配 SSE + 本地 VAD 架构）

本指南用于在实现完成后，端到端验证特性是否满足 spec.md 的成功标准。**不含实现代码**，仅给出可执行的验证场景与预期结果。

---

## 前置条件

- Windows 10 / 11 x64
- Rust 1.80+（ort 2.x MSRV 要求）
- 已 `cargo tauri dev` 启动开发版
- 已获取阶跃星辰 StepAudio API Key（Step Plan 套餐）
- 系统已有可用音频输出设备（扬声器/耳机），且为系统默认输出
- 准备一段带人声的播放内容（如 YouTube 视频、本地播客 MP3）

### 开发环境额外步骤

1. **silero-vad 模型文件**：需放置 `src-tauri/assets/silero_vad.onnx`（约 2.3MB）。
   获取方式：`pip install silero-vad`，然后从 `silero_vad/data/silero_vad.onnx` 复制到 assets 目录。模型为 silero-vad v5，state shape `[2,1,128]`，需 64-sample context 前缀。

2. **Tauri 2 权限**：`src-tauri/capabilities/default.json` 已声明 `core:event:allow-listen` 等权限。若前端报 `event.listen not allowed`，检查此文件是否被加载。

3. **代理设置**：国内开发环境需禁用代理（cargo/npm/stepfun API 直连可达）：
   ```
   HTTP_PROXY= HTTPS_PROXY= NO_PROXY="*" npm run tauri dev
   ```

## 场景 1: 首次配置到首次识别（验证 SC-006）

**目标**: 首次用户 5 分钟内完成主流程。

**步骤**:

1. 启动工具，应显示主面板（设置/历史），字幕窗未出现
2. 进入设置 → API Key，填入有效 Key，保存
3. 点击「开始字幕」
4. 在任意播放器播放带人声内容

**预期**:
- 步骤 3 后悬浮字幕窗出现，初始显示「[等待字幕...]」占位
- 步骤 4 后说话停顿 800ms 内字幕窗开始滚动显示草稿文字（partial）
- 句尾停顿后草稿归档为最终句（final）
- 主面板显示「已连接」状态

**通过判据**: 上述流程 ≤ 5 分钟完成，无需查阅外部文档。

> **延迟说明**：SSE 一次性方案下，字幕延迟 = 句尾静音 800ms（VAD 切句）+ SSE 请求往返 + 模型识别时间，典型 1-3s。连续说话时每 10s 强制提交一次。这与原 WS 流式方案的近实时延迟有本质差异。

## 场景 2: 实时延迟与稳定性（验证 SC-001、SC-002）

**目标**: 30 分钟连续识别无崩溃、无冻结、延迟达标。

**步骤**:

1. 按场景 1 启动识别
2. 播放一段 30 分钟的播客或长视频
3. 全程观察字幕滚动
4. 30 分钟后查看任务管理器中进程内存

**预期**:
- 字幕持续滚动，无明显卡顿
- 句尾停顿后草稿归档为最终句
- 连续说话时每 10s 至少有一次 final 提交
- 内存增长幅度 ≤ 初始占用的 30%
- 无崩溃、无冻结

## 场景 3: 鼠标事件穿透（验证 SC-003、FR-003）

**目标**: 字幕窗不阻塞底层播放器交互。

**步骤**:

1. 启动字幕识别，字幕窗显示在屏幕上方
2. 将播放器窗口移至字幕窗下方，使其部分被字幕窗覆盖
3. 用鼠标点击被覆盖区域的播放器控件（暂停按钮、进度条）
4. 用鼠标滚轮在覆盖区域滚动

**预期**:
- 暂停/播放正常响应
- 进度条可拖动
- 滚轮事件传递给播放器（如视频快进/音量调节）

**通过判据**: 100% 的事件穿透。

---

## 场景 4: 历史回看与检索（验证 SC-004、FR-008、FR-009）

**目标**: 会话结束后的检索能力。

**步骤**:

1. 完成一次 ≥ 30 秒的识别后，点击「停止字幕」
2. 进入历史面板
3. 确认列表中出现刚才的会话（含起止时间）
4. 点击该会话，查看完整时间轴字幕
5. 在搜索框输入刚才识别中出现的某个关键词
6. 观察检索结果

**预期**:
- 会话列表正确显示
- 会话详情按时间顺序展示所有最终字幕
- 关键词检索 ≤ 1 秒返回结果
- 命中关键词高亮，显示所属会话与时间

## 场景 5: 网络中断恢复（验证 SC-005、FR-006）

**目标**: 断连不丢音频。

**步骤**:

1. 启动识别，播放器持续播放
2. 临时断开网络（关闭 Wi-Fi 或拔网线）5 秒
3. 观察字幕窗与连接状态
4. 恢复网络
5. 观察字幕是否补齐

**预期**:
- 断网期间字幕窗显示「连接异常」状态
- 音频捕获与 VAD 切句不受影响（本地处理）
- utterance 在 channel 中堆积（上限 8 个，约 80s 音频）
- 恢复网络后 drainer 任务串行补发堆积的 utterance
- 超过 channel 容量时主任务阻塞，音频捕获暂停（背压）

> **与 WS 方案差异**：SSE 无持久连接，断网只影响提交不影响捕获。WS 方案需重连并补发缓冲帧。

## 场景 6: 默认输出设备切换（验证 SC-008、FR-014）

**目标**: 设备热切换无需重启。

**步骤**:

1. 启动识别，音频从内置扬声器播放
2. 播放过程中连接蓝牙耳机（或 USB 声卡）
3. 系统默认输出自动切换到新设备
4. 播放器音频改从新设备输出

**预期**:
- 字幕中断 ≤ 3 秒
- 工具自动重新绑定到新设备
- 无需手动重启工具

---

## 场景 7: 样式与位置定制（验证 FR-011、FR-012）

**目标**: 个性化配置生效且持久化。

**步骤**:

1. 在设置面板修改字号（如 20→28）、底板透明度（如 0.5→0.8）、文字颜色
2. 实时观察字幕窗样式变化
3. 拖动字幕窗到屏幕底部居中
4. 关闭工具
5. 重新启动工具

**预期**:
- 样式修改实时生效
- 字幕窗可拖动到屏幕任意位置
- 重启后样式与位置恢复为上次配置

---

## 场景 8: API Key 无效处理（验证 FR-013）

**目标**: 密钥问题不静默失败。

**步骤**:

1. 进入设置，清空或填入无效 API Key
2. 点击「开始字幕」

**预期**:
- 工具弹出明确提示「API Key 无效或未配置」
- 引导用户前往设置面板
- 不进行任何识别尝试

---

## 场景 9: 长时间运行日志体积（验证 SC-007、FR-018）

**目标**: 日志可控。

**步骤**:

1. 启动工具并保持识别开启 8 小时（可加速：缩短至 1 小时后按比例推算，或检查日志轮转机制）
2. 查看日志目录（`%APPDATA%\<bundleId>\logs\`）

**预期**:
- 日志文件存在轮转（多个按大小切分的文件）
- 总占用磁盘 ≤ 50 MB（8 小时口径）

---

## 场景 10: DRM 内容识别（验证 FR-016）

**目标**: DRM 内容有可见提示。

**步骤**:

1. 启动识别
2. 播放受 DRM 保护的内容（如 Netflix 视频流）

**预期**:
- 工具识别到无法捕获有效音频
- 字幕窗或通知区域显示「该内容受保护，无法识别」
- 不输出乱码字幕
